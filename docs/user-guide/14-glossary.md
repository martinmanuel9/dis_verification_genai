# Glossary

## A

**Actor Agent**
AI agent that performs analysis or generation tasks. Works independently and in parallel with other actor agents.

**Agent**
An AI model instance configured with specific role, prompt, and parameters to perform specialized tasks.

**Agent Set**
A group of agents organized to work together in a specific sequence or pattern for complex analysis.

**AI Model**
The underlying language model used for generation (e.g., GPT-4o, Llama 3.1).

**API Key**
Authentication credential for accessing cloud AI services like OpenAI.

**Arctic Embed**
Snowflake's open-source embedding model used for generating vector representations of text. US-based (Montana).

## C

**Celery**
Distributed task queue system used for background processing of long-running tasks.

**Chunk**
A segment of a document created during processing for vector storage and retrieval.

**Chunking Strategy**
Method used to divide documents (page-based, section-based, or fixed-size).

**ChromaDB**
Open-source vector database used for storing document embeddings and enabling semantic search.

**Citation**
Reference to the source document location (page number, section) for RAG responses.

**Collection**
Group of related documents stored together in ChromaDB for organized search.

**Contradiction Agent**
Specialized agent that identifies conflicts, gaps, and inconsistencies in requirements or agent outputs.

**Critic Agent**
Agent that reviews and validates outputs from actor agents, providing quality control.

## D

**Deep Analysis**
Most thorough analysis mode using 5+ agents and multiple review rounds (30-60 minutes).

**DIS**
Defense Information Systems - military and defense IT systems and networks.

**Docker**
Containerization platform used to package and run the application and its dependencies.

**Docker Compose**
Tool for defining and running multi-container Docker applications via YAML configuration.

## E

**Embedding**
Vector representation of text that captures semantic meaning, used for similarity search.

**Embedding Model**
AI model that converts text to vector embeddings (e.g., Snowflake Arctic Embed).

## F

**FastAPI**
Modern Python web framework used for the application's REST API backend.

## G

**GPT (Generative Pre-trained Transformer)**
OpenAI's family of large language models (GPT-3.5, GPT-4, GPT-4o, GPT-5).

**GPU (Graphics Processing Unit)**
Hardware accelerator that can significantly speed up AI model inference, especially for Ollama.

## H

**Hierarchical Analysis**
Multi-level agent structure where agents work in stages, with each level building on previous.

## I

**Inference**
Process of running an AI model to generate responses or predictions.

## L

**LangChain**
Framework for developing applications with language models, used for orchestration.

**LangSmith**
Optional tracing and debugging tool for LangChain applications.

**Llama**
Meta's open-source family of large language models (Llama 3.1, Llama 3.2).

**LLM (Large Language Model)**
AI model trained on vast amounts of text data to understand and generate human language.

## M

**Markdown**
Lightweight markup language used for formatting text in documents and test plans.

**MarkItDown**
Library for converting various document formats to Markdown.

**Multi-Agent System**
Architecture using multiple AI agents working together for comprehensive analysis.

**Multimodal**
AI model that can process multiple types of input (text, images), like GPT-4o.

## O

**Ollama**
Platform for running large language models locally on your own hardware.

**OpenAI**
AI research company that develops GPT models and provides API access.

**Orchestration**
Coordination of multiple agents to work together in a specific workflow or pattern.

## P

**Page-Based Chunking**
Strategy that creates one chunk per document page, preserving page structure.

**Phi**
Microsoft's efficient small language model series (Phi-3 Mini).

**PostgreSQL**
Open-source relational database used for storing application data, sessions, and metadata.

**Prompt**
Text instruction given to an AI model to guide its response or behavior.

**Prompt Engineering**
Art and science of crafting effective prompts to get desired AI outputs.

**PyMuPDF**
Python library for PDF processing, used for text and image extraction.

## Q

**Quick Analysis**
Fastest analysis mode using 2-3 agents and basic review (5-10 minutes).

## R

**RAG (Retrieval Augmented Generation)**
Technique that enhances AI responses by retrieving relevant document chunks as context.

**Redis**
In-memory data store used for caching and as message broker for Celery.

**Requirements Matrix**
Table mapping requirements to test cases, showing traceability and coverage.

**Reasoning Model**
Specialized OpenAI models (o1, o3) designed for deep logical reasoning tasks.

## S

**Score Threshold**
Minimum similarity score (0-1) for RAG results to be considered relevant.

**Section-Based Chunking**
Strategy that creates chunks aligned with document sections or chapters.

**Semantic Search**
Search based on meaning rather than keywords, using vector similarity.

**Snowflake**
Company providing Arctic Embed embedding models. US-based (Montana).

**Standard Analysis**
Balanced analysis mode using 3-5 agents with full review (15-30 minutes).

**Streamlit**
Python framework for building web applications, used for the application's UI.

**System Prompt**
Instructions that define an AI agent's role, personality, and behavior.

## T

**Temperature**
Parameter controlling randomness in AI responses (0=deterministic, 2=very creative).

**Test Card**
Individual test case with detailed procedure, prerequisites, and pass/fail criteria.

**Test Plan**
Comprehensive document describing all tests needed to verify requirements.

**Token**
Basic unit of text processed by language models (roughly 0.75 words in English).

## U

**User Prompt**
Specific instruction or question sent to an AI model for each interaction.

## V

**Vector**
Numerical representation of text in multi-dimensional space, used for similarity comparison.

**Vector Database**
Database optimized for storing and searching vectors (e.g., ChromaDB).

**Vector Embedding**
Process of converting text to vector representation using an embedding model.

**Vision Model**
AI model that can analyze and describe images (e.g., GPT-4o, LLaVa, BLIP).

## Common Abbreviations

**API** - Application Programming Interface
**CSV** - Comma-Separated Values
**DB** - Database
**DOCX** - Microsoft Word document format
**GPU** - Graphics Processing Unit
**JSON** - JavaScript Object Notation
**LLM** - Large Language Model
**PDF** - Portable Document Format
**RAG** - Retrieval Augmented Generation
**REST** - Representational State Transfer
**SQL** - Structured Query Language
**TXT** - Plain text file
**UI** - User Interface
**URL** - Uniform Resource Locator
